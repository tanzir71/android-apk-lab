Install from unknown sources

Android usually blocks APKs from outside Google Play until you explicitly allow sideloading. The exact labels vary by phone brand, but the path is usually Developer options plus the per-app unknown-source install permission.

Enable developer mode

  1. Open Settings, then About phone. On Samsung and some other phones, open Software information first.
  2. Tap Build number seven times and enter your screen lock if Android asks. This unlocks Developer options.
  3. Go back to Settings and open Developer options. If your phone exposes an install-from-unknown-sources control there, enable it for this install.
  4. Do not change unrelated developer settings. You only need enough access to let Android install an APK you downloaded yourself.

Allow the APK source

  1. Download the APK from this page on your Android phone.
  2. Open Settings, then Apps, then Special app access, then Install unknown apps. Some phones place this under Security or Privacy.
  3. Choose the browser, Files app, or download manager you used and enable Allow from this source.
  4. Open the downloaded APK and install it. After installation, turn Allow from this source off again if you want the tighter default.

APK / 04

Permissions this APK asks for

Android permissions are part of the product. Read these before installing, especially when SMS, contacts, location, Bluetooth, microphone, or background services appear.

CAMERA
Scans PromiseQR codes.
READ_CONTACTS
Looks up people when creating or importing promises.
SEND_SMS
Sends verification or reminder codes through native SMS when enabled.
BLUETOOTH / NEARBY_WIFI_DEVICES
Supports Android Nearby peer-to-peer sync.
ACCESS_FINE_LOCATION (older Android)
Required by some Android Nearby/Bluetooth discovery paths on older devices.
